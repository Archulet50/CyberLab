## Versioning 

 

CyberLab uses Semantic Versioning: 

 

- **MAJOR** version for significant platform changes. 

- **MINOR** version for new projects and features. 

- **PATCH** version for bug fixes, documentation updates, and minor improvements. 

 

Examples: 

 

- `2.1.0` → Portfolio Edition 

- `2.2.0` → New cybersecurity projects 

- `2.2.1` → Documentation improvements
