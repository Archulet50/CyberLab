# Security Policy 

 

Thank you for helping keep CyberLab secure. 

 

CyberLab is an educational cybersecurity portfolio focused on defensive security, automation, detection engineering, threat intelligence, and digital forensics. Responsible disclosure of security issues is appreciated. 

 

--- 

 

# Supported Versions 

 

CyberLab is actively developed. Security updates are applied to the latest version of the repository. 

 

| Version | Supported | 

|----------|:---------:| 

| v2.1.x | ✅ Yes | 

| v2.0.x | ❌ No | 

| v1.x | ❌ No | 

 

--- 

 

# Reporting a Security Issue 

 

If you discover a security issue within this repository, please: 

 

1. **Do not publicly disclose the issue immediately.** 

2. Open a **private GitHub Security Advisory**, if available, or contact the repository owner directly through GitHub. 

3. Include: 

   - A clear description of the issue 

   - Steps to reproduce the issue 

   - Potential impact 

   - Suggested remediation (if known) 

 

Every report will be reviewed and addressed as promptly as possible. 

 

--- 

 

# Scope 

 

This policy applies to: 

 

- Python source code 

- Project documentation 

- Configuration files 

- Repository workflows 

- Supporting scripts and utilities 

 

It does **not** apply to third-party software, operating systems, or external tools referenced by CyberLab. 

 

--- 

 

# Responsible Disclosure 

 

Please allow reasonable time for investigation and remediation before publicly discussing any reported issue. 

 

CyberLab values responsible security research and constructive collaboration. 

 

--- 

 

# Security Best Practices 

 

CyberLab is developed with the following principles: 

 

- Defensive security focus 

- Secure coding practices 

- Version control with Git 

- Clear documentation 

- Continuous improvement 

- Educational and ethical use 

 

--- 

 

# Educational Use 

 

CyberLab is intended for educational, research, and defensive cybersecurity purposes only. 

 

Users are responsible for ensuring all testing is performed on systems they own or are explicitly authorized to assess. 

 

--- 

 

# Contact 

 

Questions, feedback, and responsible security reports can be submitted through GitHub Issues or GitHub Security Advisories (where available).
